# Terra.OS — PROJECT GUIDE

## 1. OVERVIEW
**Terra.OS** is an interactive, presentation-grade simulation of a construction management SaaS platform.
Built for **Maciek**, a small-to-mid-size earthworks contractor from Dzierżoniów, this simulation demonstrates how AI can transform the traditional bidding, costing, and construction planning workflow.

## 2. CORE PROBLEM
Maciek loses money not on the excavation site, but in the office:
- Missed tenders (manual searching across BZP, TED, BK).
- Manual costing (5-15 days per tender).
- Hidden risks in bill of quantities (BOQ) vs. project drawings (margin killers).

## 3. THE SOLUTION: PIPELINE METAPHOR
The entire product is visualized as a **Shovel (Łopata)** — the universal symbol of earthworks.
- **Handle (Trzonek):** Module 1 (Input/Intake) — "Zwiad" (Tender Search).
- **Shaft (Kij):** Module 2 (Processing/Transmission) — "Kosztorysant & Silnik" (Costing & Engine).
- **Blade (Łyżka):** Module 3 (Output/Action) — "Decyzja & Mózg" (Decision & Execution).

## 4. DESIGN PHILOSOPHY
- **Industrial & Honest:** No fluff, no AI-purple gradients. Concrete, steel, neon accents.
- **Anti-Slop:** Zero template layouts. Every screen serves a purpose.
- **State-of-the-Art:** Next.js 14, Tailwind v4, shadcn/ui, Motion, Recharts.
- **Pre-Flight Ready:** Every component passes strict Taste Skill validation (WCAG AA, responsive, dark mode).

## 5. DELIVERABLES
- 43 Markdown specification files.
- Interactive simulation prototype (Next.js).
- Deployed on Vercel.
- Mock data engine with 20+ Polish tenders.
