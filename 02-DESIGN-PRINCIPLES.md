# DESIGN PRINCIPLES: ZIEMIA DESIGN SYSTEM

## 1. CORE PRINCIPLES
1. **Form Follows Function:** No decorative elements. Every pixel serves the data.
2. **Industrial Honesty:** Materials and colors reflect construction reality.
3. **Clarity Over Cleverness:** Data must be readable instantly. No hidden interactions.
4. **Anti-Template:** Zero default LLM aesthetics (no Inter, no AI-purple, no 3 equal cards).
5. **Performance First:** LCP < 2.5s, INP < 200ms. Mobile-responsive by default.

## 2. AESTHETIC FAMILY
- **Style:** Brutalist-Industrial meets Modern SaaS.
- **Mood:** Technical, precise, robust, trustworthy.
- **Keywords:** Concrete, Steel, Neon, Monospace, Grid, Blueprint.

## 3. TASTE SKILL INTEGRATION
- **DESIGN_VARIANCE:** 7 (Offset layout, asymmetric grids, varied image aspect ratios).
- **MOTION_INTENSITY:** 6 (Fluid CSS transitions, scroll-reveal on key sections).
- **VISUAL_DENSITY:** 5 (Standard app spacing, clear hierarchy, no clutter).

## 4. BANNED PATTERNS
- ❌ AI-purple gradients.
- ❌ Generic stock photos.
- ❌ "Used by Acme Corp" text logos.
- ❌ Em-dashes (`—`) anywhere. Use hyphens or periods.
- ❌ Wrapped CTA buttons.
- ❌ 3-column equal feature cards.
- ❌ Centered hero with dark mesh background.
