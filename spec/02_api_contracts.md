# spec/02 — API contracts (local REST, FastAPI)

The API is **local only** (bound to `127.0.0.1`, called by the Tauri UI in-process and by the mobile app
over LAN/secure tunnel). No public exposure. All responses JSON. All list endpoints paginate
(`?limit&cursor`). All write endpoints that cause external side-effects return `202` + an `approval_id`
(see Approval gate) instead of acting immediately.

## Conventions

- Base path `/api/v1`. Auth: desktop = none (loopback); mobile = `Authorization: Bearer <device_token>`.
- Errors use one shape:
  ```json
  { "error": { "code": "string", "message": "string", "details": {} } }
  ```
  Codes: `not_found`, `validation_error`, `conflict`, `approval_required`, `cost_cap_exceeded`,
  `engine_infeasible`, `source_unavailable`, `internal`.
- **Provenance** is included on every analytical object: `{source, doc_id, page, line_or_pos, confidence}`.
- Streaming (agents, chat) uses **SSE** (`text/event-stream`) with events `token`, `step`, `flag`, `done`, `error`.

## Module 1 — Zwiad

```
POST /ingest/run                      -> {agent_run_id}        # trigger source pull (also scheduled)
GET  /tenders?status&cpv&voiv&cursor  -> Page<TenderListItem>  # match_score desc
GET  /tenders/{id}                    -> TenderDetail          # incl. documents, analysis, discrepancies
POST /tenders/{id}/watch              -> {ok}                  # start tracker agent
POST /tenders/{id}/analyze            -> {agent_run_id}        # fetch docs + RAG + discrepancy engine
GET  /tenders/{id}/analysis          -> Analysis{summary_md, red_flags[], discrepancies[]}
POST /tenders/{id}/autofill          -> 202 {approval_id}      # Tier 2+: prepare tender forms (gated)
```

`TenderListItem`: `{id,title,buyer,cpv[],voivodeship,value_pln,deadline_at,status,match_score,match_reason}`
`Discrepancy`: `{id,kind,severity,message,provenance,axiom_code}`

## Module 2 — Kosztorysant

```
POST /tenders/{id}/estimate          -> {estimate_doc_id, estimate_owner_id}   # builds BOTH variants
GET  /estimates/{id}                 -> Estimate{variant,total_net_pln,lines[],params}
PATCH /estimates/{id}/params         <- {params:{...}} -> Estimate   # variable sidebar edits, recompute
POST /estimates/{id}/chat            -> SSE                  # chat-brain edits; emits step/flag/token
GET  /tenders/{id}/estimate/compare  -> {doc_total, owner_total, delta_pln, margin_headroom_pct}
POST /tenders/{id}/rules/check       -> RuleCheck{violations:[Flag]}   # live documentation-rule violations
# RFQ / email-broker
POST /tenders/{id}/rfq               <- {scope_desc, counterparties[]} -> 202 {approval_id}  # prepares emails
GET  /rfq/{id}                       -> RFQ{status, messages[], parsed_offers[]}
```

`Estimate.line`: `{description,unit,quantity,unit_price?,labor_pln?,material_pln?,equipment_pln?,line_total_pln,provenance}`

```
POST /tenders/{id}/engine/run        -> EngineResult
```

## Approvals (gate for every external side-effect)

```
GET  /approvals?status=pending       -> [ApprovalRequest]
POST /approvals/{id}/approve         -> {executed:true, result:{...}}   # executes the gated action + audits
POST /approvals/{id}/reject          -> {ok}
```

## System

```
GET  /health                         -> {db, ollama, bedrock_reachable, disk_free}

POST /agents/{run_id}/pause|resume|cancel

POST /backup/run                     -> {ok}
```

## Acceptance for this layer
- OpenAPI 3.1 generated from FastAPI must include every endpoint above.
- Contract tests (`tests/contract/`) assert shapes and that all side-effectful endpoints return
  `202 + approval_id` (never act inline). A test proves `/approvals/{id}/approve` is the only path that
  triggers email/dispatch/submit, and writes an `audit_log` row.
